# Restaurant Table Monitor

## Запуск проекта

1. Скачиваем проект с GitHub и переходим в локальный репозиторий
```bash
git clone git@github.com:PavelCraft/restaurant-table-monitor.git
cd restaurant-table-monitor
```

2. Скачиваем оригинальное видео в папку `video` по ссылке:
https://drive.google.com/file/d/1rYmJB13vvV96JuDUrBvlEXtoKFPWo75A/view

3. Создаём и активируем виртуальное окружение
```
python -m venv venv
source venv/bin/activate  # Linux / Mac
venv\Scripts\activate     # Windows
```

4. Устанавливаем зависимости
```
pip install -r requirements.txt
```

5. Запускаем программу
```
python main.py --video "video/видео 2.mp4" # prod-режим, более медленный, но точный
```
или 
```
python main.py --video "video/видео 2.mp4" --mode dev # dev-режим, более быстрый
```
---

## Используемое видео и стол

* Видео: `video/видео 2.mp4`
* Выбранный стол (ROI):

  ```
  x=340, y=336, w=885, h=739
  ```
* Критерий попадания:
  человек считается за столом, если центр bounding box находится внутри ROI.

---

## Логика детекции

1. Используется модель YOLOv8 (класс `person`).
2. Для каждого кадра:

   * находятся люди
   * вычисляется центр bounding box
   * проверяется попадание в ROI
3. Состояние стола:

   * `OCCUPIED` — есть человек в ROI
   * `EMPTY` — нет
4. События:

   * `EMPTY → OCCUPIED` → **стол занят**
   * `OCCUPIED → EMPTY` → **стол свободен**
5. Время события:

   * берётся напрямую из видео (`cv2.CAP_PROP_POS_MSEC`)
   * первое событие принудительно = `0 сек` для корректного старта

## Режимы работы

### prod (по умолчанию)

* YOLO на каждом кадре
* максимальная точность
* медленнее

### dev

* YOLO раз в N кадров (`frame_skip`)
* используется кэш боксов
* быстрее примерно в 10 раз
* недостаток: рамки вокруг людей двигаются с опозданием и рывками

Во время обработки видео, если предыдущие шаги выполнены без ошибок, вы увидите прогресс-бар.

## Проблемные случаи

### 1. Частичное перекрытие человека

Когда человек наклоняется к столу:

* модель может **не распознать человека**
* стол ошибочно считается свободным

<img src="https://raw.githubusercontent.com/PavelCraft/restaurant-table-monitor/ab84e91/images/head_down.png" width="500" alt="Частичное перекрытие человека">

---

### 2. Неоднозначные ситуации

Сценарий:

- семья подходит к столу
- оставляют вещи (игрушка, самокат)
- уходят на несколько минут (вероятно, в туалет)
- возвращаются, забирают вещи, ничего не заказывают и уходят

Модель **не фиксирует занятие стола**, пока нет людей в кадре.

Трактовка такой ситуации может быть неоднозначной: гости оставили вещи, но ничего не заказали.

Для определения корректного поведения системы необходимо участие **продакт-менеджера (или бизнес-аналитика)** — он сформулирует правило: считать ли стол занятым при наличии вещей без людей, если гости в итоге ничего не купили.

## Постобработка аналитики

Основная программа может фиксировать **короткие ложные посадки**:
- официант проходит мимо стола
- гость на секунду вышел из кадра
- гости оставили вещи и быстро вернулись

Для получения более точной статистики предназначен скрипт `postprocess.py`.

### Принцип работы

1. Загружает файл `output/analytics.csv`, созданный основной программой
2. Преобразует интервалы в плоский список событий (`approach` / `leave`)
3. Удаляет пары `approach → leave`, если интервал между ними меньше заданного порога
4. Восстанавливает интервалы из очищенного списка
5. Сохраняет результат в `output/analytics_clean.csv`

### Запуск

```bash
python postprocess.py
```
или 
```bash
python postprocess.py --threshold 10 --input "output/analytics.csv" --output "output/analytics_clean.csv"
```

Параметры:

* --threshold — минимальная длительность посадки в секундах (по умолчанию 10)
* --input — путь к входному CSV (по умолчанию output/analytics.csv)
* --output — путь для сохранения результата (по умолчанию output/analytics_clean.csv)

## Ограничения решения

* возможны ложные переключения состояний
* нет трекинга объектов (каждый кадр независимо)
* не учитываются оставленные вещи (игрушки, сумки, самокаты)
* нет фильтрации коротких событий на уровне основной программы (вынесено в постобработку)
* детекция по центру bbox может давать ошибки при частичном перекрытии человека
* нет разделения на гостей и сотрудников — официант, проходящий мимо стола, может быть ошибочно засчитан как гость

## Возможные улучшения

* добавление трекинга (например, DeepSORT) для учёта перемещений людей
* использование сегментации вместо bbox для более точного определения пересечения с ROI
* детекция оставленных предметов, если нужно считать стол занятым, когда гости оставили вещи и временно отошли
* интеграция фильтрации коротких событий в основной пайплайн
* автоматический подбор ROI по детекции столов
* разделение гостей и сотрудников (например, по униформе или зонам перемещения) для исключения ложных срабатываний
